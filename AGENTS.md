# s1-palette — Agent Context

## Project Overview
A React-based web application that generates color palettes from images, specifically formatted for PreSonus Studio One `.colorpalette` files. It analyzes images in-browser, extracts dominant or categorical color schemes, and generates clean, uniform, and aesthetically pleasing color ramps.

## Core Color Algorithms (in `src/App.jsx`)

1. **Non-Overlapping Category Partitioning**:
   The 360-degree HSL Hue space ($0.0$ to $1.0$) is divided into 8 distinct, mutually exclusive categories to prevent duplicate rows of color variations:
   * `red`: `[0.0, 0.05]` and `[0.95, 1.0]` (0° – 18° & 342° – 360°)
   * `orange`: `[0.05, 0.13]` (18° – 46.8°)
   * `yellow`: `[0.13, 0.24]` (46.8° – 86.4°)
   * `green`: `[0.24, 0.44]` (86.4° – 158.4°)
   * `teal`: `[0.44, 0.54]` (158.4° – 194.4°)
   * `blue`: `[0.54, 0.69]` (194.4° – 248.4°)
   * `purple`: `[0.69, 0.83]` (248.4° – 298.8°)
   * `magenta`: `[0.83, 0.95]` (298.8° – 342°)

2. **1% Minimum Pixel Threshold**:
   To prevent transition noise, compression artifacts, and tiny accents from generating full rows of colors, a category must contain at least 1% of the total analyzed pixels (minimum 15 pixels) to be activated.

3. **Per-Row Similarity Filtering & Interpolated Row Stretching**:
   To ensure the final color grid is perfectly uniform (every row has exactly `colorsPerHue` tiles), the filtering and formatting is applied per row:
   * **Filter**: Variations are generated, then filtered for lightness/saturation limits and color similarity (`deltaE` in CIELAB space).
   * **Stretch**: If the unique color count $M$ is less than the target size $K$:
     * If $M = 1$, a monochromatic ramp is generated by spacing Lightness from $0.15$ to $0.85$.
     * If $M > 1$, intermediate colors are created by interpolating H, S, L values (including circular hue wrapping) between the sorted unique colors.

## Coding Conventions
* **React**: Written in React 19 using modern functional components and hooks (`useState`, `useEffect`, `useRef`).
* **Styling**: Tailored with TailwindCSS v4. Keep design minimal, modern, and dark-mode friendly.
* **Canvas Processing**: Image analysis is done via a temporary `canvas` context. Ensure `img.crossOrigin = "Anonymous"` is set before loading to prevent CORS security errors on remote files.
* **Color Conversions**: Ensure accurate float-to-integer mapping when exporting hex. Studio One uses standard `.colorpalette` JSON structure containing an array of hex colors.
